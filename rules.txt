MINIRT SCENE CONFIGURATION RULES
--------------------------------

GENERAL STRUCTURE:
1. Each line describes exactly one object in the scene.
2. The first token in the line is the type identifier (one or two letters).
3. Object configuration lines can appear in any order.
4. One or more blank lines may appear between configuration lines.
5. All parameters for a given object must be present on its line; no parameters can be omitted.
6. Entire objects may be omitted from the file:
   - Renderable objects (sp, pl, cy, L) simply won't be rendered if missing.
   - Non-rendered unique objects (A, C) will use default values if missing.
7. If a unique object (A or C) appears multiple times, only the last occurrence is used.
8. Renderable objects can appear multiple times to create multiple instances.
9. Parameters in a line must be separated by one or more spaces or tabs.
10. No newline characters may appear inside a single object's configuration line.
11. Parameter order must strictly follow the format for that object type.
12. All numeric and vector values must respect their valid ranges.

OBJECT FORMATS:

Ambient Light:
  Format: A <ambient_ratio> <R,G,B>
  - ambient_ratio: floating-point value in range [0.0, 1.0]
  - R,G,B: integer values in range [0, 255]

Camera:
  Format: C <x,y,z> <orient_x,orient_y,orient_z> <FOV>
  - x,y,z: floating-point coordinates of the camera position
  - orient_x, orient_y, orient_z: floating-point normalized vector components, each in [-1.0, 1.0], magnitude must equal 1
  - FOV: floating-point field of view in degrees, range [0, 180]

Light:
  Format: L <x,y,z> <brightness> <R,G,B>
  - x,y,z: floating-point coordinates of the light position
  - brightness: floating-point ratio in range [0.0, 1.0]
  - R,G,B: integer values in range [0, 255]

Sphere:
  Format: sp <x,y,z> <diameter> <R,G,B>
  - x,y,z: floating-point coordinates of the sphere center
  - diameter: floating-point value
  - R,G,B: integer values in range [0, 255]

Plane:
  Format: pl <x,y,z> <norm_x,norm_y,norm_z> <R,G,B>
  - x,y,z: floating-point coordinates of a point on the plane
  - norm_x, norm_y, norm_z: floating-point normalized vector components, each in [-1.0, 1.0], magnitude must equal 1
  - R,G,B: integer values in range [0, 255]

Cylinder:
  Format: cy <x,y,z> <axis_x,axis_y,axis_z> <diameter> <height> <R,G,B>
  - x,y,z: floating-point coordinates of the center of the cylinder
  - axis_x, axis_y, axis_z: floating-point normalized vector components, each in [-1.0, 1.0], magnitude must equal 1
  - diameter: floating-point value
  - height: floating-point value
  - R,G,B: integer values in range [0, 255]
